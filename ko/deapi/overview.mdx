[**deAPI**](https://deapi.ai)는 GamerHash GPU 네트워크의 개발자 통합 레이어입니다. 80만 개 이상의 유휴 게이밍 GPU를 단일하고 주소 지정 가능한 OpenAI 호환 REST 엔드포인트로 전환하여, 별도의 인프라 구축 없이도 모든 애플리케이션과 워크플로가 AI 워크로드를 네트워크로 라우팅할 수 있게 합니다.

## 제공되는 것

- **OpenAI 호환 REST.** 이미 OpenAI SDK를 사용하는 코드에 그대로 적용 가능 — `base_url`과 API 키만 변경하면 됩니다. 동일한 chat completions, 동일한 스트리밍, 동일한 응답 형식.
- **멀티 모달 커버리지** — 텍스트, 이미지, 비디오, 임베딩, 트랜스크립션. 하나의 자격 증명으로 모든 모델 클래스 사용.
- **공식 [n8n](https://n8n.io) 노드** — API 코드를 작성하지 않고도 n8n 워크플로에서 네트워크를 호출.
- **라우팅과 페일오버 처리** — deAPI가 사용 가능한 GPU로 디스패치하고, 일시적 실패 시 재시도하며, 모델 적합도에 따라 균형을 맞춥니다.
- **비용 투명성** — 모델별 가격 책정, 실시간 사용량 대시보드, 예산 설정.
- **Stripe + 암호화폐 결제** — Stripe로 USD 결제, 또는 원하면 암호화폐로 결제.

## 누가 사용하는가

- **AI 앱 빌더** — OpenAI 대신 deAPI로 LLM 호출을 라우팅하는 Telegram 및 Discord 봇.
- **n8n 워크플로 운영자** — 저렴하고 프라이빗한 추론이 필요한 자동화 파이프라인.
- **인디 개발자** — 채팅, 이미지, 비디오 기능을 위한 MVP 백엔드.
- **Web3 프로젝트** — 온체인과 오프체인 컴퓨팅이 모두 필요한 에이전트.

2026년 초 기준, deAPI에는 **6,000명 이상의 개발자가 등록**되어 있습니다.

## 왜 OpenAI / Anthropic 대신 deAPI를 사용하는가

- **비용** — 워크로드는 그렇지 않으면 유휴 상태였을 일반 소비자용 GPU에서 실행됩니다. 단가 구조가 하이퍼스케일러 클라우드와 다릅니다.
- **오픈 소스 모델, 모델별 속도 제한 없음** — Llama 3, Qwen 3, FLUX, LTX-Video 등을 선택할 수 있습니다. 독점적 제약이 없습니다.
- **프라이버시** — 콘텐츠는 디스패치된 후 폐기됩니다. 네트워크는 정산 후 프롬프트나 결과물을 보관하지 않습니다.
- **실제 사용자에게 돌아가는 실제 돈** — 본인 지출이 하이퍼스케일러의 마진이 아닌 GPU를 운영하는 사람들에게 자금을 공급합니다.

## 알아둘 트레이드오프

- **지연 시간** — 일반 소비자용 GPU는 단일 데이터센터에 함께 위치하지 않습니다. 첫 토큰 지연은 중앙화 클라우드보다 길며, 특히 콜드 스타트 워크로드에서 그렇습니다.
- **모델 가용성** — 카탈로그는 큐레이션되어 있고 꾸준히 성장하지만 무한하지는 않습니다. 틈새 또는 독점 모델은 사용할 수 없을 수 있습니다.
- **용량** — 피크 수요 시 큐 시간이 늘어날 수 있습니다. 표준 속도 제한이 적용됩니다.

## 아키텍처

deAPI 요청은 다음과 같이 흐릅니다.

```
클라이언트
   │  HTTPS (OpenAI 호환 요청)
   ▼
deAPI 게이트웨이
   │  인증, 빌링, 라우팅
   ▼
GamerHash 디스패처
   │  모델 + VRAM + 지역 매칭
   ▼
GPU 노드 (AI App을 실행 중인 기여자)
   │  추론
   ▼
결과는 동일한 경로로 반환
```

API를 사용하기 위해 GPU 측을 알 필요는 없습니다. 그러나 다음은 알아두는 것이 좋습니다. deAPI를 통해 흐르는 모든 달러는 **플랫폼 인프라**, **GHX 매입·소각**, **GUSD로 직접 기여자 보상** 사이에서 분배됩니다.
